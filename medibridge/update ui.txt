
MediBridge — X-Ray Diagnosis Module Integration
Product Requirements Document (PRD)

1. Objective
Extend MediBridge with an AI-powered X-ray diagnosis feature allowing users to:
- Navigate from Health Analyzer to X-ray page
- Upload X-ray images
- Analyze using Hugging Face models with fallback mechanism
- Generate diagnosis, precautions, and medication suggestions
- Display results in UI matching existing illness cards

Implementation must be fully additive and must not modify existing CSS or functionality.

2. Navigation Requirements
Add an "X Ray" button next to the Analyze button on the Health Analyzer page.
On click navigate to /xray-diagnosis using React Router.

3. Frontend Requirements
Create:
frontend/src/pages/XrayDiagnosis.jsx
frontend/src/pages/XrayDiagnosis.css (scoped only)

UI must match MediBridge theme:
- Dark gradient background
- Glassmorphism cards
- Neon accents
- Responsive layout

Features:
- Image upload (drag & drop + file picker)
- Image preview
- Analyze button with loading state
- Results card styled like illness result UI

4. Backend Requirements
Create route:
backend/routes/xrayRoutes.js

Endpoint:
POST /api/xray/analyze

Flow:
Receive image → Validate → Convert buffer → Call HF models with fallback → Generate report → Return JSON

5. Hugging Face Models (Best Suited with Fallback)

Primary Model (Most Accurate Multi‑Disease Chest Detection):
- torchxrayvision/densenet121-res224-chex

Secondary Model (Modern Transformer Vision Model):
- codewithdark/vit-chest-xray

Fallback Model (Lightweight Backup):
- ianpan/chest-x-ray-basic

Fallback Mechanism:
System must attempt prediction in the following order:

1. DenseNet CheXNet model
2. ViT Chest X-ray model
3. Basic Chest model

If one fails due to timeout, rate limit, or invalid response → automatically switch to next.

All models must use the same Hugging Face API key.

Important:
Image models do NOT consume tokens like LLMs.
They are request‑based inference systems.

6. Optional Explanation Model (Max Tokens High Intelligence)

If OpenRouter key is available, use:

meta-llama/llama-3.3-70b-instruct:free

Purpose:
- Medical explanation
- Diagnosis report generation
- Precautions and medication suggestions

This model supports very large context (~65k+ tokens depending provider).
